## 黑屏排查（根因：WebGL context 泄漏）
- 现象：用户报"黑屏"（之前是"一直在加载"→"黑屏"）。服务端 200、各模块转译 200，排除服务挂/语法错。
- 根因：`PixelBlast.jsx` 的 useEffect cleanup 里有一行 `if (threeRef.current && mustReinit) return;` 是**反向 bug**——在 React StrictMode（dev 默认双挂载：mount→unmount→mount）下，第一次卸载因 `mustReinit===true` 直接 return，**旧的 WebGL context 没被 dispose/forceContextLoss 释放**。dev server 反复 HMR/重启累积泄漏，超过浏览器 WebGL context 上限(~16) 后 `new THREE.WebGLRenderer()` 抛错，且无错误边界接住 → 整棵 React 树崩溃 → body 暗色背景=黑屏。
- 修复（3 处）：① 删掉 cleanup 那行错误 `return`，统一在 cleanup 真正 dispose + forceContextLoss 释放 context；② `PixelBlast`/`SpecularCard` 的 WebGL 初始化整体包 `try/catch`，创建失败只禁用背景、不拖垮整页；③ 新建 `src/components/ErrorBoundary.jsx` 并在 `main.jsx` 包裹 `<App/>` 作根级兜底（任何组件 throw 都降级为可读提示而非纯黑）。
- 验证：5 个改后模块 dev 转译均 200，无语法错。
- 关键教训：浏览器侧已泄漏的 context **HMR 局部更新不会释放，必须整页硬刷新（Ctrl+Shift+R）**才能清除超限 context 让修复生效。WebGL 组件务必在 useEffect cleanup 释放 context；WebGL 创建必须 try/catch 防整页崩溃。

## 黑屏二次排查（根因：`onVisibility` 作用域错误）
- 现象：用户刷新后仍黑屏，错误边界显示 `onVisibility is not defined`。
- 根因：上一步把 `onVisibility` 定义在 `useEffect` 的 `if (mustReinit)` 分支内，但 cleanup 函数总是引用它。当依赖变化但不触发 reinit 时，`mustReinit===false`，else 分支不会定义 `onVisibility`，而 cleanup 闭包访问到未定义变量直接抛 ReferenceError → ErrorBoundary 兜底。
- 修复：将 `const onVisibility = () => {...}` 与 `document.addEventListener('visibilitychange', onVisibility)` 提到 `useEffect` 函数体顶部，确保每次 effect 实例都有定义；cleanup 仍用同一引用移除监听器。HMR 重编译后 5 个关键模块均 200。
- 提醒：修复已生效，用户需硬刷新页面清除浏览器端旧脚本缓存与已泄漏的 WebGL context。
